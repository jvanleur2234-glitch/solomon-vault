# Background: Solomon OS Paste-to-Agent Orchestration

## Vision
Build the "OpenClaw Convo Messenger" equivalent for Solomon OS:
- User pastes any app URL (or zo.pub link) into Telegram
- Zo reads the URL, analyzes the app
- Spawns a Hermes agent or Russell Tuna worker to take actions inside it
- Russell Tuna (or Hermes Worker) executes tasks via PinchTab browser

## What We Have
- Zo + Hermes = AI brain
- Russell Tuna = Telegram AI worker (streaming, free via Ollama)
- PinchTab = headless Chrome with HTTP API (port 9888, token: solomon123)
- browser-use = AI browser automation
- Camofox = anti-detection headless browser (port 9377)
- Hermes has 94 skills

## What's Missing (The Gap)
1. **Telegram message handler** that reads URLs and triggers agent workflow
2. **Orchestration layer** — connects Telegram -> Zo -> Hermes/Russell -> PinchTab
3. **Context injection** — when Russell Tuna gets a URL, it needs the full Solomon OS context
4. **Approval flow** — AI actions should be reviewable before execution

## Architecture
```
User sends URL in Telegram
  -> Zo (this agent) receives it
  -> Zo reads the URL (browser or read_webpage)
  -> Zo determines: Hermes skill? Russell Tuna task? Browser action?
  -> If browser action: call PinchTab API (port 9888)
  -> If agent task: spawn Hermes or Russell Tuna sub-agent
  -> Results sent back to user via Telegram
```

## Implementation Steps
1. Create a Telegram bot command `/agent [url]` that triggers the workflow
2. Use PinchTab for browser automation (already running on port 9888)
3. Russell Tuna can be wired to receive tasks via the Zo API
4. Hermes skills handle specialized tasks

## Quick Win
- First iteration: Zo receives URL, reads it, summarizes it, proposes actions
- User approves, Zo executes via PinchTab
