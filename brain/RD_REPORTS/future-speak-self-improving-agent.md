## What It Does
A controlled self-modification engine for AI agents. Reads own code, proposes targeted changes with autogenerated diffs, requires explicit human approval before write or hot-reload. Framework-agnostic, zero dependencies.

## Key Features
- Framework-agnostic, zero dependencies
- Human-in-the-loop approval required
- Safe diff generation
- Read/list allowed without approval; write/modify only after approval
- Hot-reload upon approval
- Derived from Agent Friday project

## Safety Model
- Reading and listing: free, no approval needed
- Proposing changes: requires approval
- Writes and hot-reloads: only after explicit human consent
- "No silent modification policy"

## For Solomon OS / Hermes
- Safety-first self-improvement pattern — good model for Hermes self-evolution
- Hot-reload capability for Hermes Agent without restart
- Can be bolted onto existing agent frameworks
