## What It Is
A contemplative multi-agent decision-making framework inspired by Quaker business practices. Instead of racing to answers, it convenes multiple perspectives to seek clarity and unity. Designed for weighty decisions — architecture choices, ethical trade-offs, competing approaches.

## Key Skills
- **deliberation:discernment** — Internal voices surface tensions without papering over them
- **deliberation:clearness** — Multi-agent committee for deep parallel analysis (code reviews, architecture)
- **deliberation:gathered** — User joins alongside agent voices (user has stake/perspective)

## Core Principles
| Principle | Meaning |
|-----------|---------|
| Sense of the meeting | Discern unity, don't count votes |
| Speaking once | Each perspective speaks once, then listens |
| Silence | Space lets insights emerge |
| Standing aside | "I wouldn't, but I won't stop you" |
| Blocking | Rare — only core principle violations |
| Way opens | Clarity emerges, don't force decisions |

## Comparison to Quorum
| Aspect | Deliberation | Quorum |
|--------|--------------|--------|
| Style | Contemplative, slow | Structured, 7-phase |
| Voices | 3-5 specialized | Multi-model council |
| Speed | Slower, deeper | Faster, systematic |
| Best for | Ethical decisions, trade-offs | Complex multi-domain analysis |
| User involvement | High (gathered mode) | Medium (human-in-loop option) |
